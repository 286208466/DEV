### 1. 备份原 WordPress 网站
A. 备份文件

通过 FTP/SFTP（如 FileZilla）或 SSH 下载整个 WordPress 根目录（通常位于 /var/www/html/wordpress 或 /home/username/public_html）。

```bash
cd /usr/local/nginx/html/cn
tar -czvf wordpress_backup.tar.gz cn/

#
tar -czvf wordpress_main_backup.tar.gz wordpress_main/
#
tar -czvf wordpress_ru_backup.tar.gz ru/
#
tar -czvf wordpress_ro_backup.tar.gz ro/

```

包含所有文件（wp-content、wp-config.php 等）。

B. 备份数据库

使用 mysqldump 导出数据库（替换 db_name、db_user）：

```bash
mysqldump -u root -p wordpress_cn > wordpress_cn_db.sql

#
mysqldump -u root -p wordpress_main > wordpress_main_db.sql
#
mysqldump -u root -p wordpress_ru > wordpress_ru_db.sql
#
mysqldump -u root -p wordpress_ro > wordpress_ro_db.sql
```

输入密码后，会生成 wordpress_db.sql 文件。

### 2. 在新服务器上准备环境
A. 安装 LAMP/LEMP

确保新服务器已安装：

- Web 服务器（Apache/Nginx）

- MySQL/MariaDB

- PHP（版本需匹配原服务器）

B. 创建数据库

登录 MySQL：

```bash
mysql -u root -p
```

创建新数据库和用户（同原名或新名）：

```bash
CREATE DATABASE new_db_name;
CREATE USER 'new_db_user'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON new_db_name.* TO 'new_db_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3. 上传文件到新服务器

A. 上传 WordPress 文件

通过 FTP/SFTP 或 scp 上传备份的 wordpress_backup.tar.gz 到新服务器的网站目录（如 /var/www/html）：

```bash
scp wordpress_backup.tar.gz user@new_server_ip:/var/www/html
```

解压：

```bash
tar -xzvf wordpress_backup.tar.gz
```

B. 导入数据库

上传 wordpress_db.sql 到新服务器。

导入数据库：

```bash
mysql -u root -p wordpress_cn < wordpress_db.sql

mysql -u root -p wordpress_main < wordpress_main_db.sql
```

### 4. 修改 WordPress 配置
A. 更新 wp-config.php

编辑新服务器的 wp-config.php，确保数据库信息正确：
```bash
define('DB_NAME', 'wordpress_cn');
define('DB_USER', 'root');
define('DB_PASSWORD', 'password');
define('DB_HOST', 'localhost');
```

B. 替换域名（如需）

如果域名变更，需更新数据库中的旧域名：

使用 sed 快速替换（备份数据库后操作）：

```bash
sed -i 's/old_domain.com/new_domain.com/g' wordpress_db.sql
```

或在 WordPress 数据库执行 SQL：
```bash
UPDATE wp_options SET option_value = 'http://www.chinpee.com' WHERE option_name = 'siteurl' OR option_name = 'home';


select * from wp_options where option_name = 'siteurl' OR option_name = 'home';
```

### 5. 检查权限和配置
A. 设置文件权限

```bash
chown -R nginx:nginx /var/www/html/cn
chmod -R 777 /var/www/html/cn
```

B. 配置 Web 服务器

Apache：确保 DocumentRoot 指向 WordPress 目录。

Nginx：检查 server 块配置是否正确。

### 6. 测试访问
临时修改本地 hosts 文件测试（如域名未解析）：

```bash
new_server_ip new_domain.com
```

访问 http://new_domain.com，检查是否正常显示。
