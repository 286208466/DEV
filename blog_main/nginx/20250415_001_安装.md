在 CentOS 7 上通过 Nginx 源码包（.tar.gz） 安装，可以获取最新版本或自定义编译选项。以下是详细步骤：

### 1. 安装依赖环境


备份原有的yum源文件，更新yum源为阿里云

```bash
curl -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

yum clean cache

yum makecache

yum update
```

```bash
yum -y install gcc-c++
yum install -y pcre pcre-devel
yum install -y zlib zlib-devel
yum install -y openssl openssl-devel
```

:::tip

必须安装的依赖：

- gcc：编译工具

- pcre-devel：正则表达式支持

- zlib-devel：Gzip 压缩支持

- openssl-devel：HTTPS 支持

:::

### 2. 下载 Nginx 源码包

访问 Nginx 官网 获取最新稳定版链接（如 nginx-1.25.3）：

```bash
wget https://nginx.org/download/nginx-1.25.3.tar.gz
tar -zxvf nginx-1.25.3.tar.gz
cd nginx-1.25.3
```

### 3. 编译安装

基础编译（默认配置）

```bash
./configure --prefix=/usr/local/nginx

make && make install
```

默认安装路径：/usr/local/nginx/

自定义编译（推荐）

```bash
./configure \
--prefix=/usr/local/nginx \
--with-http_ssl_module \
--with-http_stub_status_module \
--with-pcre
make && sudo make install
```

:::tip

常用模块：

--with-http_ssl_module：启用 HTTPS

--with-http_stub_status_module：启用状态监控

--with-pcre：增强正则支持

:::

### 4. 启动 Nginx

```bash
# 检查配置文件语法
sudo /usr/local/nginx/sbin/nginx -t

# 启动
sudo /usr/local/nginx/sbin/nginx

# 重启/停止
sudo /usr/local/nginx/sbin/nginx -s reload  # 重载配置
sudo /usr/local/nginx/sbin/nginx -s stop   # 停止
```
### 5. 设置系统服务（方便管理）
创建服务文件：

```bash
sudo vi /etc/systemd/system/nginx.service
```

写入以下内容：

```bash
[Unit]
Description=The NGINX HTTP and reverse proxy server
After=syslog.target network.target remote-fs.target nss-lookup.target

[Service]
Type=forking
PIDFile=/usr/local/nginx/logs/nginx.pid
ExecStartPre=/usr/local/nginx/sbin/nginx -t
ExecStart=/usr/local/nginx/sbin/nginx
ExecReload=/usr/local/nginx/sbin/nginx -s reload
ExecStop=/bin/kill -s QUIT $MAINPID
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 6. 验证安装
检查版本：

```bash
/usr/local/nginx/sbin/nginx -v
```

访问测试：

浏览器输入服务器 IP，看到 Welcome to nginx! 即成功。

### 7. 配置防火墙

```bash
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

### 常见问题

- 端口冲突

如果 80 端口被占用（如 Apache），先停止相关服务：

```bash
sudo systemctl stop httpd
```

- 模块缺失

若编译时报错（如缺少 SSL 模块），确保安装了 openssl-devel 并重新 ./configure。

- 自定义路径

修改 --prefix=/your/path 指定安装目录，后续所有路径需同步调整。

### 卸载源码安装的 Nginx

```bash
sudo rm -rf /usr/local/nginx/
sudo rm /etc/systemd/system/nginx.service
sudo systemctl daemon-reload
```
